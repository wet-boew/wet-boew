Technique Information

Technology

Guidelines Reference

Success Criterion 1.3.1 (Info and Relationships)

Technique Title

Summaries a Data Column Group in a Data Table

Applicability

Use with column data group on data table when summary information is present.

User Agent Issues

Not Available

Description of Technique

The objective of this technique is to separate the data column group with their associative data summary column group. This concept is similar as the "tfoot" definition and the row group summary.

A data table need to have column header cell to define a column summary group.

The column group level is calculated based on the highest row that would identify the data group with the summary group. The level of the column data group is calculated based on their associate column summary group.

By default, if there is only one row of column header cell, the first data column group is at level 1. If a table contain for example 3 group (excluding the column header group), the first one is a data group at level 1, the seconde one is a summary group at level 1 and the third one is a summary group at level 0. No other group can exist after the summary column group at level 0 is reached.

For the best tabular data representation is to be consistant in the column heading structure.

A column cell heading that cover a data group and a summary group is called a group header cell. Each group header cell start on the begin of a column group and end to his associated summary or before.

Normally a data column group is larger (Contain more cell) than his associated summary group
