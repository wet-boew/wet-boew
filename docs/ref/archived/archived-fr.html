Contenu archivé

  • Examples pratiques
  • Questions ou commentaires?

La Norme sur la facilité d’emploi du Web nécessite un avis de contenu archivé pour clairement aviser les utilisateurs que la page a été archivée sur le Web.

Aperçu

Responsable du projet : Stéphane Bérubé (@berubs)

Objet

Ajoute un avis d'archivage à une page Web.

Modèle de page web archivée

Avantages

  • La convivialité de cette approche a fait l’objet d’essais auprès des utilisateurs
  • Respecte l’article 6.1.5 de la Norme sur la facilité d’emploi du Web
  • Permet aux utilisateurs d’identifier le contenu archivé lors de la consultation d’un site à partir d’un moteur de recherche en incluant le mot « ARCHIVÉ » dans la page titre (les éléments <h1> et <title>)
  • Une plus grande utilisation de cette tendance à l’échelle du gouvernement du Canada permettra aux utilisateurs d’apprendre à identifier le contenu archivé dans tout site du gouvernement du Canada
  • Comprend un lien vers le contenu de remplacement, le cas échéant
  • Permet aux utilisateurs d’établir que le contenu a été archivé lorsqu’ils arrivent à un lien ancré, et non au haut de la page, en incluant une amélioration progressive qui indique que la page Web a été archivée lorsque l’utilisateur fait dérouler la page.

Quand l’utiliser

Cette approche est obligatoire pour les pages Web du gouvernement du Canada qui ont été archivées en ligne.

Une page Web peut être archivée en ligne si :

  • elle est conservée à des fins de référence, de recherche ou de tenue de documents
  • ne sera pas modifiée ou mise à jour après la date
  • est entreposée dans un référentiel numérique.

Par exemple, une nouvelle politique qui en remplace une ancienne, les versions précédentes de rapports, des événements qui ont déjà eu lieu (comme des conférences)

Quand ne pas l’utiliser

  • La page Web comprend du contenu à jour
  • Plutôt que de supprimer le contenu redondant, insignifiant et dépassé (RID)
  • Plutôt que de ne pas changer la page Web pour respecter la nouvelle Norme sur la facilité d’emploi du Web (WCAG 2.0)

Mise en œuvre

Pages HTML

  1. Ajoutez "ARCHIVED - " / "ARCHIVÉE -" à la page titre et à l’élément de contenu <h1>.
  2. Ajoutez le code suivant après l’élément de contenu <h1> (corrigeant le cheminement de l’image et les catégories de classe grille au besoin et remplaçant le message d’accueil par du texte plus pertinent à la page archivée)

Pages qui ne sont pas HTML

Pour les pages archivées qui ne sont pas HTML (p. ex. PDF), ajouter une page interstitielle :

  • HTML qui utilise cet élément (tel que décrit dans la section sur les pages HTML) et indiquer que le document que vous avez consulté a été archivé
  • Si vous pouvez modifier le document qui n’est pas en HTML, ajoutez également :
    • boîte d’avis d’archivage
    • « ARCHIVÉ » dans le titre et
    • ARCHIVÉ dans le lien

Développement

Le code pour archiver du contenu se trouve à différents endroits dans le dossier source de la BOEW :

  • js/workers/archived.js - compris dans le code JavaScript pour archiver du contenu
  • js/sass/includes/_archived.scss - comprend la SCSS pour archiver du contenu
  • js/images/archived/ - comprend les images pour archiver du contenu

Enjeux connus

  • La solution décrite dans le Guide de mise en œuvre de la Norme sur l'accessibilité des sites Web n’a pas été mise à jour pour tenir compte de l’approche ayant fait l’objet d’essais
  • Le lien de la page Réduire le contenu RID ne fonctionne plus depuis le retrait de la bibliothèque de modèles
