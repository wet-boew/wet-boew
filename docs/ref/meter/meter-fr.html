Aperçu

Chef du projet : Paul Jackson (@pjackson28)

Objet

L'élément meter de HTML5 affiche une mesure scalaire dans une gamme connue. Étant donné que certains navigateurs ne prennent pas en charge cette fonctionnalité à l'origine, ce correctif émule la même fonctionnalité en utilisant HTML générique et WAI-ARIA.

Raison d’être

Les navigateurs qui ne prennent pas en charge l'élément progress ignore l'élément. Par conséquent, rien est affiché et son contenu n'est pas exposé à des technologies d'assistance. Ce correctif assure que l'élément meter est visuellement convertie en une barre coulissante et exposé aux technologies d'assistance en ajoutant WAI-ARIA.

Mise en œuvre

Pour utiliser le correctif, l'élément meter doit être utilisé. Dans les cas où le navigateur ne soutien pas l'élément meter le correctif est automatiquement chargé.

Code d'exemple

Exemples pratiques

Normal

<meter min="20" value="65" max="80">75 % (normal)</meter>

Trop haut

<meter min="0" value="90" max="100" high="80">90 % (trop haut)</meter>

Trop bas

<meter min="100" low="120" value="115" max="200" high="180">15 % (trop bas)</meter>

Développement

Le compteur est dépendent sur le soutien natif pour l'élément meter de HTML5 et le correctif pour meter.

  • js/polyfills/datalist.js - contient le code JavaScript pour le correctif pour meter
  • js/sass/includes/_datalist.scss - contient le CSS pour le correctif pour meter
