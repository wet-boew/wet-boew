Accessibility responsibility breakdown (WCAG 2.0)

  • Working examples
  • Questions or comments?

Overview

Project Lead: Denis Boudreau (@dboudreau)

Purpose

Provides a breakdown of WCAG 2.0 accessibility responsibilities throughout the Web production chain.

Benefits

  • Helps to identify the accessibility responsibilities for each role in the Web production chain
  • Provides a tabbed-based interface for quickly focusing on the relevant role in the Web production chain
  • Conforms to WCAG 2.0 AA
  • Uses WAI-ARIA to enhance accessibility
  • Progressive enhancement approach

Recommended usage

  • Identifying the accessibility responsibilities for each part of the Web production chain

Implementation

Migrate the *.html files to the Web site's layout and design.

Development

Accessibility responsibility breakdown is dependent upon Coopérative AccessibilitéWeb articles (MIT license).

Known Issues

  • Certain Web production chains may use different terminology
  • Certain Web production chains may divide up roles and responsibilities differently
