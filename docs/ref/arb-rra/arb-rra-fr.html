Répartition des responsabilités d'accessibilité (WCAG 2.0)

  • Examples pratiques
  • Questions ou commentaires?

Aperçu

Responsable du projet : Denis Boudreau (@dboudreau)

Objet

Une répartition des responsabilités associées aux critères de succès pour WCAG 2.0 sur l’ensemble de la chaîne de production Web.

Avantages

  • Permet d’établir les responsabilités liées à chacun des rôles de la chaîne de production Web
  • Présente une interface basée sur des onglets pour trouver rapidement le rôle pertinent de la chaîne de production Web
  • Respecte WCAG 2.0 AA
  • Repose sur WAI-ARIA pour améliorer l’accessibilité
  • Approche d’améliorations progressives

Utilisation recommandée

  • Établir les responsabilités en lien avec l’accessibilité pour chacun des éléments de la chaîne de production Web

Mise en œuvre

Migrer les dossiers *.html selon la présentation et le format du site Web.

Élaboration

La distribution des responsabilités pour l’accessibilité dépend de [http://accessibiliteweb.com/fr/accessibilite/repartitions/wcag-2-0.html articles de la Coopérative AccessibilitéWeb] (Licence MIT)

Enjeux connus

  • Certaines chaînes de production Web peuvent utiliser des terminologies différentes
  • Certaines chaînes de production Web peuvent diviser les rôles et les responsabilités de différentes façons
