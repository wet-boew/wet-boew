Overview

Project lead: Jeff Gordon (@jeffdavidgordon)

Purpose

Adapts WET for use with SSI.

Benefits

  • Helps to minimize the duplication of content and code (such as titles and dates)
  • Provides central control of the left and right columns
  • Improves consistency by separating content from the template code
  • Automates the breadcrumb trail
  • Conforms to WCAG 2.0 AA
  • Uses WAI-ARIA to enhance accessibility

Minimum requirements

  • dist and demos folders from WET 3.1.x
  • Server Side Includes installed on a Web server
  • Apache 2.x Web Server

Implementation

  1. Make sure that you have the WET 3.1.x dist and demos folders in your root directory.
  2. Merge the dist and demos folders from the master branch of the wet-boew/wet-boew-ssi repository with the WET 3.1.x dist and demos folders:
    1. /dist/shtm is a folder with your global include files
    2. /dist/theme-xxxx are folders with theme specific include files
