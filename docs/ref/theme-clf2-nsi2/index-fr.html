Thème de la NSI 2.0

Aperçu

Chef du projet : Madeleine Daigle (@maddydaigle)

Avantages

  • Les premières tâches ont été essayées avec les utilisateurs : contactez-nous, la recherche, retourner à la page d'accueil, à propos de nous, rester connexes par les médias sociaux, naviguer le site, lire les avis, et basculer la langue officielle de choix.
  • Conformes à la Norme sur l'accessibilité des sites Web et la Norme sur la facilité d'emploi des sites Web
  • Conformes à WCAG 2.0 AA
  • Recours aux balises sémantiques HTML5 et à la spécification WAI-ARIA pour accroître l’accessibilité
  • Regroupe le gros de la navigation sous le domaine du contenu
  • Conformité aux critères HTML5 et XML

Utilisation recommandée

Quand utiliser

Utiliser pour la transition vers la dernière version de la Boîte à outils de l'expérience Web avant de passer au thème de la facilité d'emploi GC. Lorsque cela est possible, les ministères doivent passer au thème de la facilité d'emploi GC en même temps que la mise à niveau vers la dernière version de la Boîte à outils de l'expérience Web afin de réduire les coûts.

Quand ne pas utiliser

Ce thème peut seulement être utilisé pour les sites Web du gouvernement du Canada.

Contenu

Le contenu devrait se fixer sur les premières tâches des utilisateurs. Les utilisateurs ont tendance à naviguer par le zone de contenu premier et les menus deuxième. Ils cherchent des mots-clés qui les aideront à rencontrer leur besoin d'informations. Écrire dans la voix active et les verbes d'usage. Les utilisateurs scruteront souvent les informations distinctes - comme les liens. Chaque lien doit être descriptif et représentatif du contenu à lequel il mène. Ceci est surtout prononcé quand les utilisateurs accèdent au site via un appareil mobile.

Mise en œuvre

Instructions générales

  1. Utiliser \theme-clf2-nsi2\*.html pour créer vos pages Web.
  2. Facultatif : Créer et installer un our plusieurs fichers CSS (pour le CSS personnalisé)
  3. Facultatif : Créer les liens aux nouveaux fichiers CSS entre les commentaires suivants : 
    < !-- CustomScriptsCSSStart -->
< !-- CustomScriptsCSSEnd -->
  4. Corriger les URLs des *.css, *.js et *.gif dans les fichiers *.html
  5. Remplacer le titre de la page (entre <title> et </title>,), le titre du site et le titre du contenu (entre <h1 id="wb-cont"> et </h1>)
  6. Remplacer les valeurs des métadonnées
  7. Corriger les liens dans le pied de page

Instructions spécifiques aux pages de contenu (cont*.html)

  1. Corriger les liens de la barre de menu
  2. Corriger la signature institutionelle
  3. Corriger le titre du site
  4. Corriger le fil d'Ariane
  5. Corriger les liens du menu latéral (si le menu existe)
  6. Corriger le valeur pour Date modified/Date de modification

Instructions spécifiques aux pages d'entrée (sp-pe*.html)

  1. Corriger les liens anglais et français

Instructions spécifiques aux pages de messages du serveur (serv*.html)

  1. Remplacer les messages en anglais et français

Développement

Le code pour le thème de la NSI 2.0 se trouve dans plusieurs places dans le dossier source de la BOEW :

  • theme-clf2-nsi2/js/theme.js - contient le JavaScript du thème de la NSI 2.0
  • theme-clf2-nsi2/sass/theme.scss - contient le CSS pour les pages de contenu du thème de la NSI 2.0
  • theme-clf2-nsi2/sass/theme-ie.scss - contient le CSS spécifique à IE7/8 pour les pages des contenu du thème de la NSI 2.0
  • theme-clf2-nsi2/sass/theme-serv.scss - contient le CSS pour les pages de messages du serveur du thème de la NSI 2.0
  • theme-clf2-nsi2/sass/theme-serv-ie.scss - contient le CSS spécifique à IE7/8 pour les pages de messages du serveur du thème de la NSI 2.0
  • theme-clf2-nsi2/sass/theme-sp-pe.scss - contient le CSS pour les pages d'entrée du thème de la NSI 2.0
  • theme-clf2-nsi2/sass/theme-sp-pe-ie.scss - contient le CSS spécifique à IE7/8 pour les pages d'entrée du thème de la NSI 2.0
  • theme-clf2-nsi2/sass/includes/ - contient les fichiers à inclure pour créer les fichies CSS du thème de la NSI 2.0
  • theme-clf2-nsi2/images/ - contient les images du thème de la NSI 2.0

Problèmes connus

  • Les styles des CSS ne seront pas entièrement uniformes d’un fureteur à l’autre puisque plusieurs interpréteront différemment la syntaxe des CSS.
  • L'essai de facilité d'emploi est nécessaire pour les styles dans la système styles in the CSS grid system.
